# UI Flow

Current active flow:
- `index.html` shows the menu and non-game screens.
- `js/game.js` runs gameplay and result triggers.
- the canvas menu / font compare experiment is archived.
- active source of truth for the main menu is `index.html`
- do not revive a second parallel main-menu implementation
- canvas stays limited to gameplay, HUD, warnings, and combat overlays

Документ показывает все основные переходы между экранами первой версии UI.

## Основной поток

`Первый запуск`  
→ `Главное меню`  
→ `Игра`

## Поток игры

`Игра`  
→ `Пауза`  
→ `Настройки`  
→ `Главное меню`

`Игра`  
→ `Экран результатов`

## Поток паузы

`Пауза`  
→ `Продолжить`  
→ `Игра`

`Пауза`  
→ `Настройки`

`Пауза`  
→ `Главное меню`

## Поток главного меню

`Главное меню`  
→ `Играть`  
→ `Игра`

`Главное меню`  
→ `Рейтинг`

`Главное меню`  
→ `Настройки`

`Главное меню`  
→ `Титры`

## Поток результатов

`Экран результатов`  
→ `Повторить`  
→ `Игра`

`Экран результатов`  
→ `Главное меню`

`Экран результатов`  
→ `Рейтинг`

`Экран результатов`  
→ `Continue`  
→ `Игра`

## Поток рейтинга

`Рейтинг`  
→ `Назад`
  
→ экран, с которого открыт `Рейтинг`

## Поток настроек

`Настройки`  
→ `Назад`
  
→ экран, с которого открыты `Настройки`

## Поток титров

`Титры`  
→ `Назад`
  
→ `Главное меню`

## Правило первого запуска

- `Первый запуск` показывается только один раз.
- После первого прохождения пользователь должен попадать сразу в `Главное меню`.
