# UI Foundation

Status:
- accepted as the active UI foundation for the project
- panel-based, not canvas-based
- no longer a speculative UI kit draft

Accepted direction:
- `panel.png` is the common container for non-game screens
- the background remains visible behind the panel
- content sits inside the panel
- metal buttons remain the shared button style
- `BoldPixels` is the UI font family

Core rules:
- keep the interface dark, matte, and compact
- use cyan for normal labels and red only for danger or click-confirmation
- avoid oversized decorative blocks when the content is simple
- keep panel sections readable first, decorative second

What this document is for:
- a short reminder of the active UI language
- a place to keep the current accepted rules before the next UI session

Current UI notes:
- the settings and results screens are already on the panel path
- the controls screen still needs a better composition pass
- the results screen still needs a typography pass
- temporary experiments and screenshot outputs should not stay in the active tree once a screen is accepted
